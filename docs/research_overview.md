# 项目总说明

本文档用于让协作者在不通读代码的情况下，快速理解项目的研究目标、数据来源、处理流程、核心指标和报告生成逻辑。

## 1. 项目定位

本项目不是单纯的“风险识别系统”，也不是让大语言模型直接读取原始工程文件的文本生成器。它更准确的定位是：

**一个面向 TBM 施工报告自动化生成的数字孪生与大语言模型协同框架。**

系统先把 TBM 施工数据、超前地质预报和掌子面素描组织成一个统一、可解释、可约束的轻量化数字孪生状态，再由大语言模型生成正式施工报告。

## 2. 原始输入

### 2.1 TBM 日运行 CSV

主要包含：

- 时间字段：运行时间
- 里程字段：`chainage` 或等价里程列
- 状态字段：掘进状态
- 施工参数：推力、推进速度、给定速度、刀盘扭矩、刀盘转速、贯入度
- 气体字段：`CO2`、`H2S`、`SO2`、`NO2`、`NO`、`CH4`

### 2.2 地质证据 PDF

包括：

- `TSP` 超前地质预报
- `HSP` 超前地质预报
- 掌子面 / 洞身素描

这些资料本身已经包含外部地质结论。系统的任务不是替代专家重新判断，而是把这些资料结构化后与 TBM 数据对齐。

## 3. 总体处理链路

1. 清洗 TBM CSV，统一时间和里程
2. 解析地质 PDF，沉淀为标准化证据记录
3. 将地质证据按里程挂接到 TBM 逐点数据
4. 计算区段级 `GRS / RAI / GRCI`
5. 构建数字孪生状态摘要
6. 生成日报、时段报告和 Agent 问答结果

## 4. CSV 处理逻辑

### 4.1 基础工况分段

系统将 TBM 运行划分为：

- 停机
- 过渡
- 稳定掘进
- 异常扭矩

这一步为整体施工节奏分析和停机统计提供基础。

### 4.2 隐含施工状态识别

系统进一步用多参数聚类识别施工状态，用于解释：

- 高负载低速
- 低负载快速推进
- 稳定推进
- 状态切换和效率差异

### 4.3 气体分析

系统会对气体监测数据做统计分析，纳入报告中的安全状态部分。

### 4.4 常规环级停顿候选识别

为了降低正常工序停顿对异常分析的干扰，系统新增了常规环级停顿候选识别。它会结合：

- 停顿持续时间
- 停顿前后是否恢复工作
- 里程是否接近环级周期性位置
- 停顿期间里程漂移是否很小

识别结果不会直接删除停顿，而是用于后续 `RAI` 的停机惩罚折减。

## 5. PDF 解析与证据库

系统对 `TSP / HSP / 素描` 分别采用规则解析，提取：

- 报告编号
- 起止里程
- 围岩等级
- 裂隙、破碎、风化、稳定性
- 出水、掉块、变形
- 风险标签

解析结果统一转为标准化证据记录，并同步保存到 `SQLite`。

## 6. 地质-施工融合

系统以统一里程轴为桥梁进行融合：

- TBM 数据原本按时间记录
- 地质资料原本按里程描述

通过将 TBM 样本映射到里程轴，再按里程命中地质证据，系统得到逐点的地质状态变量，例如：

- 综合围岩等级
- 综合灾害标签
- 出水、掉块、变形标记
- 活跃来源数
- 证据强度
- 不确定性

## 7. 当前算法版本

### 7.1 GRS：区段地质关注度表征

当前版本的 `GRS` 不再使用主观工程权重，而是：

1. 对以下分量做归一化
   - 围岩等级
   - 灾害标签
   - 出水
   - 掉块
   - 来源支撑强度
2. 对归一化后的分量做等权平均

因此，当前 `GRS` 的角色是：

**区段地质关注度表征**

而不是灾害概率。

### 7.2 高斯衰减平滑

为减弱固定 `10m` 区段切割带来的边界跳变，系统在逐点 `GRS_base` 之后引入了按里程的高斯衰减平滑。

这意味着：

- 地质影响不会在区段边界处瞬时归零
- 相邻区段之间的关注度变化更加连续
- 更符合围岩渐变和刀盘响应延续的物理直觉

### 7.3 RAI：施工响应异常度表征

当前版本的 `RAI` 基于 `Isolation Forest` 构建。

输入特征包括：

- 推力
- 扭矩
- 推进速度
- 刀盘转速
- 贯入度

系统先计算逐点异常分数，再聚合为区段级 `RAI`。同时保留停机、效率和多参数一致性等解释信息，因此 `RAI` 既是数据驱动的，又保留工程解释性。

### 7.4 停机异常折减

若某些停顿被识别为常规环级停顿候选，系统会对 `row_stop_anomaly` 做折减，而不是把这些停顿完全删除。这样能降低正常停顿对 `RAI` 的误报。

### 7.5 GRCI：地质-施工耦合验证

`GRCI` 保留了原有的耦合验证思路，重点评估：

- 同步耦合
- 滞后耦合
- 响应变化
- 多参数一致性
- 来源支撑修正

因此，`GRCI` 的作用不是再次做风险预测，而是识别：

**哪些区段既有地质关注支撑，又有施工响应印证。**

这类区段最适合在报告中重点展开。

## 8. 数字孪生状态

本项目中的数字孪生不是三维几何模型，而是轻量化状态孪生。它至少同步表达：

- 时间状态
- 位置状态
- 运行状态
- 地质状态
- 安全状态
- 前方提示状态

它的作用是为报告生成提供统一、结构化、可约束的状态输入。

## 9. LLM 报告生成

大语言模型并不直接读取原始 `CSV + PDF`，而是读取系统先整理好的结构化摘要，包括：

- 工况分段
- 基础统计
- 施工状态与效率
- 当前掌子面摘要
- 前方地质融合分析
- 区段耦合分析
- 数字孪生状态摘要
- 历史对比摘要
- 气体分析

系统通过提示词严格约束模型：

- 不能把“关注”写成“已发生”
- 不能把当前掌子面和前方预测混写
- 不能把所有停机都写成异常受阻
- 不能把边界两侧区段写成不合理的绝对跳变

## 10. Agent 问答

除报告生成外，系统还提供会话式 `Supervisor Agent` 问答入口。它的角色是：

- 调用现有分析工具
- 复用数字孪生状态和历史记忆
- 支持连续追问和上下文承接

它是分析入口增强模块，不是论文主算法本体。
